## 项目说明

本项目实现了“学生信息与考场管理系统”，主要功能包括学生信息查询、随机点名、考场安排表生成、准考证文件生成。程序使用 `Student` 类保存学生信息，使用 `ExamSys` 类封装系统功能，并按 `main.py`、`exam_system.py`、`student.py` 进行模块化组织。

## 运行方法

```bash
python3 main.py
```

运行后根据菜单输入功能编号：

```text
1. 查询学生信息
2. 随机点名
3. 生成考场安排表
4. 生成准考证文件
0. 退出系统
```

## 2. 核心 Prompt 迭代记录

初代 Prompt：请帮我写一个学生信息管理系统，能查询学生、随机点名、生成考场安排和准考证。

AI 生成的问题/缺陷：初版思路容易把全部代码写进一个文件，而且异常处理不够完整；随机点名如果输入字母、0 或超过总人数，程序可能直接报错或输出不友好。

优化后的 Prompt：请按 `main.py`、`exam_system.py`、`student.py` 三个文件组织代码，必须定义 `Student` 类和 `ExamSys` 类；程序只能使用标准库中的 `os` 和 `random` 模块；随机点名必须用 `try-except` 处理非数字、小于等于0、超过总人数等情况；文件读写要使用 UTF-8，并给出友好错误提示。

优化结果：代码结构变得更清晰，菜单、查询、点名、考场安排和准考证生成都被封装在 `ExamSys` 类中，输入异常也能被统一处理。

## 3. Debug 与异常处理记录

报错类型/漏洞现象：在测试准考证功能时，如果用户没有先执行“生成考场安排表”，程序无法知道学生座位顺序，可能导致准考证内容为空。

解决过程：我检查了 `generate_admission_tickets` 的逻辑，增加了判断：如果内存中没有考场安排，则先尝试读取已有的 `考场安排表.txt`；如果文件也不存在，就自动调用 `generate_exam_arrangement` 生成安排表。这样即使用户直接选择功能4，程序也能正常生成准考证。

## 4. 人工代码审查 (Code Review)

下面代码是随机点名功能的核心逻辑，我逐行加上中文说明，证明我理解它的执行过程。

```python
def random_roll_call(self):
    try:
        count_text = input("请输入需要点名的学生数量：").strip()  # 读取用户输入，并去掉首尾空格
        count = int(count_text)  # 尝试把输入转换成整数，如果输入字母会进入 except
        if count <= 0:  # 人数不能是0或负数
            raise ValueError("人数必须大于0")
        if count > len(self.students):  # 抽取人数不能超过学生总人数
            raise ValueError("人数不能超过学生总人数")
    except ValueError:
        print("输入人数无效，请输入1到" + str(len(self.students)) + "之间的整数。")
        return  # 输入不合法时直接结束本次功能，避免程序崩溃

    selected_students = random.sample(self.students, count)  # 从学生列表中随机抽取且不重复
    print("本次随机点名结果：")
    index = 1
    for student in selected_students:
        print(str(index) + "." + student.name + " " + student.student_id)  # 按题目格式输出姓名和学号
        index += 1
```
