## 项目说明

这个项目是按第二次作业要求写的“学生信息与考场管理系统”。我把代码拆成了三个文件：`student.py` 放学生对象，`exam_system.py` 放主要功能，`main.py` 只负责启动程序。

名单文件是 `人工智能编程语言学生名单.txt`。因为作业要求只允许使用 `os` 和 `random`，所以我没有用 `csv` 之类的库，读取名单时自己处理了空格分隔和逗号分隔两种情况。

## 运行方法

```bash
python3 main.py
```

运行后根据菜单输入功能编号：

```text
1. 查询学生信息
2. 随机点名
3. 生成考场安排表
4. 生成准考证文件
0. 退出系统
```

## 2. 核心 Prompt 迭代记录

初代 Prompt：请帮我写一个学生信息管理系统，能查询学生、随机点名、生成考场安排和准考证。

AI 生成的问题/缺陷：第一次得到的方案更像一个“能跑的脚本”，有些代码直接写在一个文件里；另外随机点名只考虑了正常数字，没有认真处理 `abc`、`0`、超过总人数这些输入。

优化后的 Prompt：请按 `main.py`、`exam_system.py`、`student.py` 三个文件组织代码，必须定义 `Student` 类和 `ExamSys` 类；程序只能使用标准库中的 `os` 和 `random` 模块；随机点名必须用 `try-except` 处理非数字、小于等于0、超过总人数等情况；文件读写要使用 UTF-8，并给出友好错误提示。

我自己修改的地方：我把文件读取单独放进 `load_students`，菜单循环只负责判断功能编号；随机点名里把三种非法输入都统一放到 `ValueError` 处理，这样输出比较稳定，也不会让程序直接中断。

## 3. Debug 与异常处理记录

报错类型/漏洞现象：我测试时发现一个顺序问题：如果用户一打开程序就直接输入 `4`，这时候还没有执行过“生成考场安排表”，内存里的 `exam_arrangement` 是空列表。

解决过程：我没有让功能4直接报错，而是在 `generate_admission_tickets` 里加了一个判断：先看内存中有没有安排表；没有的话，再看目录里有没有 `考场安排表.txt`；如果文件也没有，就自动调用 `generate_exam_arrangement`。这样用户先点 4 也能正常得到准考证。

## 4. 人工代码审查 (Code Review)

下面这段是随机点名功能。我选它做代码审查，是因为这里既有输入转换，也有边界判断，比较能体现异常处理。

```python
def random_roll_call(self):
    try:
        count_text = input("请输入需要点名的学生数量：").strip()  # 先拿到字符串，去掉多余空格
        count = int(count_text)  # 这里可能失败，比如用户输入 abc
        if count <= 0:  # 0 人或负数没有实际意义
            raise ValueError("人数必须大于0")
        if count > len(self.students):  # random.sample 不能抽超过列表长度的人数
            raise ValueError("人数不能超过学生总人数")
    except ValueError:
        print("输入人数无效，请输入1到" + str(len(self.students)) + "之间的整数。")
        return  # 本次点名结束，回到菜单，不让程序崩掉

    selected_students = random.sample(self.students, count)  # sample 会保证不重复
    print("本次随机点名结果：")
    index = 1
    for student in selected_students:
        print(str(index) + "." + student.name + " " + student.student_id)  # 按题目示例输出
        index += 1
```

## 结果案例

### 功能1：查询学生信息

输入学号 `25361077`，程序输出：

```text
学生完整信息：
姓名:范哲齐
学号:25361077
```

输入不存在学号 `99999999`，程序输出：

```text
未找到该学号对应的学生，请检查输入是否正确。
```

### 功能2：随机点名

我测试了三类错误输入：`abc`、`0` 和 `100`。这三个输入都会进入异常处理，输出：

```text
输入人数无效，请输入1到20之间的整数。
```

输入 `3` 时，程序会输出三名不重复学生。因为这里用了随机函数，每次人名可能不一样，例如：

```text
本次随机点名结果：
1.范哲齐 25361077
2.陈思源 25361082
3.唐睿 25361096
```

### 功能3：生成考场安排表

程序在根目录生成 `考场安排表.txt`，每行格式为：

```text

3,罗嘉琪,25361093
```

### 功能4：生成准考证文件

程序在根目录生成 `准考证` 文件夹，并创建 `01.txt`、`02.txt` 等文件。示例：

